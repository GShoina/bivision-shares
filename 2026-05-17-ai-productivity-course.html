🏢 Bivision Context

Claude = proposals, reports, client emails. ChatGPT = market research (browsing). Gemini = inside Google Docs. Knowing which tool = right task → 3× speed per deliverable without extra cost.

🌍 Think Bigger

These tools change how you learn anything. Ask Claude to explain a new concept at 3 different depth levels. Use ChatGPT to challenge your own business decisions. The meta-skill is knowing when to trust and when to verify the AI.

👤 Individual Exercise
1A — Prompt Engineering Practice
  1. Pick any real task you do at work (write email, summarize report, plan meeting agenda)
  2. Version A: basic prompt ("write an email to a client"). Version B: RISEN framework (Role + Input + Steps + Expectation + Narrowing)
  3. Send both to Claude. Compare. What changed?
  4. Now send your best version to ChatGPT AND Gemini. Which won for this task type?
  5. Save your winning prompt as a template — you'll reuse this exact structure
👥 Team Exercise
1B — Team Prompt Competition (1h, 3+ people)
  1. Everyone picks the same topic: "Write a LinkedIn post about Bivision's BI solution for Georgian manufacturers"
  2. Each person writes a different prompt and sends to Claude. Anonymous — don't show your prompt.
  3. Share all outputs (not prompts). Team votes: which output is best?
  4. Reveal the prompts. Analyze: what made the winning prompt better?
  5. Build a shared "Prompt Library" doc with the best patterns found

🌍 Think Bigger

Agents are the boundary of current AI. Right now they fail unexpectedly, hallucinate tool calls, get stuck in loops. Understanding their limits IS the skill — knowing when to trust an agent unsupervised vs when human-in-the-loop is mandatory. The managers who thrive will be those who design agent boundaries, not those who let agents run wild.

👤 Individual
8A — Agent Opportunity Mapping
  1. List 5 weekly tasks: find info → process it → produce output (report, email, update)
  2. Score each: (a) how often? (b) how consistent? (c) how long? (d) need creativity?
  3. Top score = best agent candidate. Build it in Claude + MCP or Make.com
  4. Run it once supervised. Watch for: what breaks? Where does it need human judgment?
  5. Calculate: time saved × 50 weeks = annual hours reclaimed. Is it worth 3h to build?
👥 Team
8B — Agent Design Workshop (2h, whole team)
  1. Each person presents their top agent candidate from Exercise 8A (5 min each)
  2. Group vote: which one creates the most value for the company if built well?
  3. Design session: map out the winning agent's architecture — Trigger → Steps → Tools → Output → Human checkpoint
  4. Assign owner: who builds it? Timeline? How do we measure success?
  5. Define failure mode: what happens if the agent does something wrong? What's the rollback plan? This is the most important step — agents fail in unexpected ways.
