Security Headers — Ready-to-Paste .htaccess Block

Add to /home/bivision/public_html/.htaccess right after "# END WordPress" marker. Rollback: remove the 8 lines between markers + LiteSpeed will auto-purge cache. cPanel File Manager backup: right-click .htaccess → Download before edit = full rollback file.

# BEGIN Security Headers
<IfModule mod_headers.c>
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
    Header always set X-Frame-Options "SAMEORIGIN"
    Header always set X-Content-Type-Options "nosniff"
    Header always set Referrer-Policy "strict-origin-when-cross-origin"
    Header always set Permissions-Policy "camera=(), microphone=(), geolocation=(), interest-cohort=()"
    Header unset X-Powered-By
    Header always unset X-Powered-By
</IfModule>
# END Security Headers

Testing after deploy:

  1. Visit securityheaders.com/?q=bivision.ge → expect grade A or A+
  2. curl -sI https://bivision.ge/ | grep -i "strict-transport" → expect HSTS header present
  3. Click around site + verify no broken iframe/admin embeds

Rollback procedure:

  1. cPanel File Manager → public_html → .htaccess → Edit
  2. Delete lines between # BEGIN Security Headers and # END Security Headers
  3. Save
  4. LiteSpeed Cache → Toolbox → Purge All

Risk: HSTS is sticky — once browsers receive it, they enforce HTTPS for up to 1 year. If cert breaks later, site becomes inaccessible. Mitigation: start with max-age=300 (5 min) for 24h test, then bump to 31536000 (1 year) only after stability confirmed.
