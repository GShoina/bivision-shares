P1 - High

BUG #3 - BiRetail video tags missing width/height (13 videos, 0 with dims) FACT

P1
Verified on
/saas-products/biretail/
Symptom
Cumulative Layout Shift (CLS) - videos load with intrinsic 0x0, hero section collapses until video loads
HTML
<video autoplay loop muted playsinline preload="none"><source src="BiRetail-SaaS.webm"><source src="BiRetail-SaaS.mp4"></video>
Root cause
render_block gif-to-video filter at functions.php:1437 reads width/height from original img tag. If source img had no width/height attr (likely after 2026-05-15 re-upload), video output also lacks them.
Why BiRetail worse
BiRetail has 13 videos vs 1-2 on other pages — most damage visible there.
Fix A - functions.php patch (default dims fallback). In gif-to-video filter (line ~1437), add fallback when width/height match missing: 
if (!isset($wm[1]) || !isset($hm[1])) {
    $is_hero = strpos($all, "product-hero") !== false;
    $w_val = $is_hero ? 1280 : 800;
    $h_val = $is_hero ? 720  : 600;
} else {
    $w_val = $wm[1];
    $h_val = $hm[1];
}
$w = " width=\"" . $w_val . "\"";
$h = " height=\"" . $h_val . "\"";
Fix B (simpler): re-save BiRetail page in Gutenberg block editor - WP auto-populates width/height from media library metadata. Test single video first.
