3. სავალდებულო 8-ველიანი pre-check — სანამ წერილს დაწერ

თუ ეს 8 ველი არ შეავსე — წერილი არ დაიწეროს.
1. Recipient Role: CEO / CFO / Sales Director / Project Director
2. Company type: developer (ბინები) / construction contractor / holding / mixed
3. Trigger/Signal: რა event-ი გვაქვს? (tender win / new project announced / CFO hire / LinkedIn signal / cold — no signal)
No signal = opening უფრო მძიმე insight-ით უნდა იწყებოდეს
4. One Pain: ერთი მთავარი ტკივილი (cash flow / project P&L / budget control / debtors / sales pipeline)
მხოლოდ ერთი — ორი pain = წერილი მძიმდება
5. Consequence/Cost: რა ბიზნეს-ზარალი ან რისკი ჩანს ამ pain-ისგან? (დრო / cash gap / გვიანი გადაწყვეტილება / გადახარჯვა)
ეს ველი commercial argument-ს ქმნის — გარეშე = ფაქტის ჩამოთვლა, არა argument
6. Product Angle: BiFinance / Direct Cash Flow / Budget vs Actual / ERP integration / Debtors
7. Proof allowed: IG Development Holding ✓ / ERP integrations / Bank API / "სამშენებლო კომპანიები ქართულ ბაზარზე"
8. CTA type: reply / 15-min call / example dashboard / "2 მაგალითი გამოგიგზავნოთ?"

4. სტრუქტურა — ყოველი წერილი

Subject: 3–6 სიტყვა. კონკრეტული pain ან კითხვა. არა "Bivision გთავაზობთ".

Consequence (1–2 წინადადება): რა ჯდება ეს pain — დრო, cash risk, გვიანი ხილვა. Commercial argument.

Bivision relevance (1–2 წინადადება): კონკრეტულად რას ახერხებს — არა "BI dashboard", არამედ "project P&L / cash position / budget gap" ხილვა.
