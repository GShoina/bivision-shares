2. Challenge points — 7 independent (3 mandatory, 4 advisory)

🔴 Mandatory M1: Bivision's own schema declares inLanguage: en-US

Prompt's "Adaptation notes for non-English markets" warns: "Schema markup language attribute must match content language (inLanguage field)". Wire evidence on bivision.ge homepage right now: 
"@type":"ImageObject","@id":"https://bivision.ge/#logo",
"inLanguage":"en-US"
Prompt creator hasn't applied own advice. If Bivision distributes the prompt without fixing its own schema first, credibility gap = real reputation risk (matches Gurafa's Side A risk #5 in self-challenge). Required mitigation: Fix Bivision schema inLanguage + HTML lang="ka-GE" + og:locale ka_GE BEFORE public release. Bivision = proof-of-concept.

🔴 Mandatory M2: Citability score 0-10 — methodology absent

Prompt #1 outputs "citability score 0-10" without specifying computation. Different LLM runs = different scores. Cross-time + cross-competitor comparison broken without pinned formula. Required mitigation: Define explicit formula. Example: 
citability_score = (queries_cited_top3 / total_queries) × 10
weighted by position (1=10, 2=7, 3=5)
averaged across N≥3 runs over 7 days (engine drift control)

🔴 Mandatory M3: "+30% in 30 days" target — unsourced (memory feedback_no_fabrication.md violation)

Validation gate Mini-tier states: "target +30% in 30 days". No source cited. Per Apr 26 memory rule "no fabrication" — invented number = anti-pattern. Required mitigation: Either cite empirical source for +30% expectation OR replace with "lift target = empirical, set after baseline established + N=3 measurements over 7 days".

🟡 Advisory A1: Variable count = 7 = first-run friction

{SITE_URL}, {BRAND}, {ICP}, {CORE_OFFER}, {LANGUAGE}, {COMPETITORS}, {KEY_QUERIES} — 7 variables before any prompt runs. {KEY_QUERIES} alone = 10 well-crafted AI-search queries. Non-SEO user friction high → abandonment risk. Suggested: Pre-fill 3 archetypal examples (SaaS B2B, e-commerce, agency) as adapt-and-go templates.

🟡 Advisory A2: AI engine drift — single-run baseline = snapshot, not trend

Claude/Perplexity/ChatGPT update training + crawl frequency. Same query different week = different result. Single audit = snapshot. Suggested: Add measurement protocol: N=3 runs over 7 days, average citation rate, before declaring baseline. Re-measure cycle quarterly per validation expiration rule.

🟡 Advisory A3: Competitor "owns query because of X signal" — correlation ≠ causation

Prompt #4 attributes ownership to schema/named-clients/year-facts. Causation may be domain authority, backlink profile, age. Tactic transfer risky if root cause misdiagnosed. Suggested: Add disclaimer: "ownership reasons are hypotheses; verify by overlaying domain authority + backlink profile + age before applying competitor tactic".

🟡 Advisory A4: EEAT scoring 0-10 — rubric absent

Prompt #3 asks "0-10 per dimension + evidence". Without rubric anchors, scoring = opinion. Reproducibility broken. Suggested: Provide anchors: 
Experience: 0=no client examples
              5=2-3 client mentions
              10=year-anchored named case studies + outcome metrics
Same for Expertise / Authoritativeness / Trust.
