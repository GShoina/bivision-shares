GEO/SEO Master Prompt — Portable Claude Skill Pack

10 prompts · Site-agnostic · Copy-paste runnable in Claude / ChatGPT / Perplexity · Author: Gurafa (Bivision) · 2026-04-26 · Free to share
What this is. A portable, project-agnostic prompt pack for GEO (Generative Engine Optimization — citation in AI-search) + classic SEO. Apply to any site / brand / product. Each prompt is self-contained: trigger condition, input variables, paste-text, expected output.

Why GEO matters. AI-search (Claude, Perplexity, ChatGPT, Gemini) is replacing classic Google for B2B research. If your brand isn't AI-citable, prospects researching via AI won't find you — even if classic SEO is healthy.

How to use. Pick the prompt by stage (Audit → Optimize → Verify), fill the {VARIABLES}, paste into Claude. Output goes to disk or CMS. Re-run quarterly per the validation expiration rule.

Stage 1 — Audit (4 prompts)

1. AI-citability baseline audit

You are a GEO analyst. Run a citability audit for {BRAND} ({SITE_URL}).

For each of these 10 queries: {KEY_QUERIES}
- Simulate the response a user would get from Claude / Perplexity / ChatGPT.
- Note: is {BRAND} cited? In top-3? With what context? Or absent?

Output a table:
| Query | Cited? Y/N | Position | Context (1 line) | Citation source URL if any |

End with: 3 patterns observed (e.g., "{BRAND} cited only when query includes country name"), 3 gap hypotheses, and a citability score 0-10.

2. Schema.org markup audit audit

Fetch {SITE_URL} (or paste HTML). Audit current schema.org markup.

Report:
1. Schema types present (Organization, Service, Product, FAQ, Article, …) — list each with completeness %
2. Schema types missing but recommended for {CORE_OFFER}
3. Errors / warnings (use Schema.org validator logic)
4. Top-3 ROI fixes ranked by AI-citability impact

For each missing/incomplete schema: provide ready-to-paste JSON-LD with placeholders filled from {BRAND} / {CORE_OFFER} / {ICP}.

3. CORE-EEAT content quality scoring audit

Score the top-5 pages of {SITE_URL} on CORE-EEAT framework:
- Experience (first-hand evidence: dates, named clients, case specifics)
- Expertise (depth, technical accuracy, named experts)
- Authoritativeness (citations, mentions, partnerships)
- Trust (transparency, contact info, security signals)

Per page: 0-10 score per dimension + 1-line evidence + top-1 fix.

End with: which 1 page to fix first for max AI-citability lift, and what to add (declarative facts with year, named clients, specific outcomes).

4. Competitor AI-citation comparison audit

For {COMPETITORS} vs {BRAND}, run the same 10 queries: {KEY_QUERIES}.

Output table:
| Query | {BRAND} cited? | Competitor 1 | Competitor 2 | … | Winning citation pattern |

End with:
- Which competitor "owns" which query bucket and why (specific signal: schema, named clients, year-anchored facts, etc.)
- 3 gaps where {BRAND} can leapfrog with low effort
- 1 query where {BRAND} should NOT compete (let competitor own; differentiate elsewhere)

Stage 2 — Optimize (4 prompts)

5. Topic cluster + content map optimize

Build a topic cluster map for {BRAND} targeting {ICP}, language {LANGUAGE}, around {CORE_OFFER}.

Output:
1. 3 pillar topics (broad, evergreen)
2. 5-7 sub-topics per pillar (specific, query-matched)
3. Internal linking pattern (pillar ↔ sub, sub ↔ sub where relevant)
4. Top-10 priority pages to write/optimize (rank by AI-citability lift × business relevance)

For each priority page: 1-line H1, 3 H2 sections, 1 declarative fact-with-year to include.

6. AI-friendly Q&A extraction optimize

From {SITE_URL} content + {ICP} pain points, generate 20 question-answer pairs that an AI would use to cite {BRAND}.

Format per Q&A:
- Question: as user would phrase it in {LANGUAGE}
- Short answer: 1-2 sentences, declarative, fact-anchored (year, named entity, specific number)
- Source page: which page on {SITE_URL} backs this answer

Group by topic. Mark which 5 should become FAQ schema markup (highest AI-citation likelihood).

7. Schema markup JSON-LD generator optimize

Generate complete schema.org JSON-LD for {SITE_URL}, including:
1. Organization (with sameAs: LinkedIn, Facebook, YouTube)
2. Service (per {CORE_OFFER} variant)
3. FAQ (use top-5 Q&A from prompt #6)
4. Article (template for blog posts, with author, datePublished, dateModified)
5. BreadcrumbList (template for category pages)

Output: 5 JSON-LD blocks, each ready to paste in <head>.

Validate mentally against Schema.org spec; flag any required field I'd need to fill manually.

8. AI-citable copy rewrite optimize

Rewrite this page/section: [PASTE CURRENT COPY]

Target audience: {ICP}, language: {LANGUAGE}, brand: {BRAND}.

Apply these AI-citation rules:
1. Replace vague claims with declarative facts (add year, entity, number)
2. Add 1+ named client/case per main section
3. Use Q&A subheadings where natural (matches AI-search pattern)
4. Eliminate buzzwords (synergy, cutting-edge, innovative)
5. Each paragraph ≤ 3 sentences (AI-extractable)
6. End with 1 declarative one-liner an AI could quote standalone

Output: rewritten copy + diff summary (what changed and why per rule).

Stage 3 — Verify (2 prompts)

9. Internal link + canonical structure audit verify

Audit {SITE_URL} for:
1. Orphan pages (no internal links pointing in)
2. Canonical URL consistency (any duplicate / conflicting canonicals?)
3. Internal anchor text — generic ("click here") vs descriptive (target keyword)
4. Hub-spoke pattern integrity (do pillars link to all sub-topics? do sub-topics link back?)
5. Outbound link authority (linking out to authoritative sources increases AI-citation likelihood)

Output: top-10 fixes ranked by impact × effort. For each: file/page + before → after.

10. PDF report builder + 30-day priority action list verify

Synthesize all prior 9 prompt outputs for {BRAND} into one executive report:

Sections:
1. Executive summary (1 paragraph + 1-line citability score before/target)
2. Top-3 wins (what's working — keep)
3. Top-3 critical gaps (what's broken — fix first)
4. 30-day action plan: 10 ranked tasks with owner, effort estimate, expected impact
5. Measurement plan: what to re-measure at day 30 to verify lift
6. Appendix: schema JSON-LD blocks, Q&A library, rewritten copy samples

Format: clean markdown ready for PDF export. Audience: non-technical CEO/owner.

Usage protocol

  1. Run all 4 audit prompts first → establish baseline
  2. Pick top-3 gaps → run relevant optimize prompts (5-8)
  3. Implement outputs into CMS / codebase
  4. Wait 14-30 days for AI-search re-crawl
  5. Re-run prompt #1 → measure citability lift
  6. Quarterly: re-run #4 (competitor comparison) + #10 (PDF report)
Validation gate (per Validation Master Prompt v2 — Mini tier):
  • Build: implement top-3 gap fixes (1-2 days)
  • Measure: baseline citability score from prompt #1, target +30% in 30 days
  • Pre-mortem (Side B — cost of not doing): competitor first-mover lock in narrow AI-search vocabulary; positioning credibility leak; growing AI-search share gap (2027+ trend)

Adaptation notes for non-English markets

  • Run prompt #1 with queries in both primary {LANGUAGE} and English — AI-search may pull from English knowledge base even for native-language queries
  • Schema markup language attribute must match content language (inLanguage field)
  • For low-resource languages (e.g., Georgian, Armenian, Azerbaijani), declarative facts with year + entity carry extra weight (training data sparse → distinctive signals dominate)
  • If {BRAND} serves multiple language markets — run audit per language separately, do not blend
