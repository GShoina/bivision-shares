3. CTA audit — product pages (BiFinance, BiRetail)

2"მოითხოვე დემო" CTA = JS popup form, English-language, no Calendly (HIGH)

3 "მოითხოვე დემო" buttons per product page → all link to #lead-popup (anchor + JavaScript open). Popup contents:

  • Title: "Request a demo" (English)
  • Subtitle: "Please enter your name and number, we will contact you shortly" (English)
  • Form fields: Your name / Company / Phone number / Email (all English)
  • Button: "Request" / "Cancel"
  • Submit: inline AJAX (no `action` attr) — handled by WP custom JS via `lead_form_nonce`
  • Backend: stores in WP DB; HubSpot sync = manual / via owner workflow

This explains the HubSpot status="closed" bug: WP form submission → WP DB → manual HubSpot import → contact lands in HubSpot with default deal-stage "closed" or no stage at all. Auto-status corruption native to non-integrated form.

Impact: Georgian visitor sees English popup = trust friction (1.5-2x bounce vs language-matched form). Plus form-not-Calendly = extra step vs ad-direct strategy.

Fix options (priority):

  1. Calendly inline embed in popup OR button → direct link. Eliminate WP form. Owner-strategy aligned (everyone → Calendly).
  2. Translate form to Georgian if popup retained. Trust + conversion lift 1.5-2x.
  3. Direct HubSpot integration if form retained — Forms API, 1-click setup, eliminates manual sync.

  • Dead clicks: 0%
  • Rage clicks: 0%
  • Excessive scroll: 0%
  • Quick-back (clicked + immediately left): 0%

Translation: when a visitor lands on bivision.ge, the site itself doesn't break or confuse them. They navigate normally. The funnel collapse is upstream — visitor does NOT find a conversion path because (a) Calendly missing, (b) Demo CTA hidden in JS popup, (c) form-language mismatch. Once visitor finds a CTA and clicks, the experience is clean.

Sample size 3 sessions = small. Larger sample needed post-relaunch.
