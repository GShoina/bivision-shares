1. გაკვეთილი (session lesson)

  1. Label ≠ substance. "Priority" / "ready" / "validated" memory-დან ნასესხები სიტყვაა. Disk verify + dev interview + prototype საჭიროა, ამის გარეშე = blackbox.
  2. Classic terms ≠ domain fit. JTBD "schema / mapping / testing / errors / back-and-forth" = textbook template. Bivision-ს შესაძლოა სხვა terminology ჰქონდეს — დომეინი არ ვიცი → კითხვარიც არ ვიცი.
  3. Distribution არ არის ნაგულისხმები. SSOT ფაილი ≠ team awareness. ყოველი shared-core update ითხოვს explicit broadcast + ACK.
  4. Self-challenge missed. Peer-challenge (Viktor/Mentari) + owner review ყოველი artifact-ზე pre-ship, არა post.
  5. Theory-loop default = agent trap. "Recommend" / "Let's explore" / "consider" = words for when concrete step missing. Cure = force BML + JTBD + Pre-mortem.

2. Validation Master Prompt — 3 Framework

Slot 1 — Build-Measure-Learn (Eric Ries, Lean Startup)

Trigger: claim რომ "X-ს ვაშენებთ / ვრეკომენდებთ"
Build: minimum working artifact (prototype / test output / measurement) — 1-3 დღე max. არა slide, არა analysis doc.
Measure: ციფრი / binary outcome pre-defined. Baseline vs target წინასწარ.
Learn: pass → რა ვიცით? fail → რა ვიცით? ორივე სცენარი actionable უნდა იყოს.
Output: docs/BML_<idea>_<YYYY-MM-DD>.md

Slot 2 — Jobs-To-Be-Done Interview (Clayton Christensen)

Trigger: claim რომ "user/dev/client-ს X სჭირდება"
5-Q script (15 წთ):
  1. Context — რა routine-ს აკეთებ? რამდენი დრო?
  2. Pain — რომელი ნაწილი იწელება? კონკრეტული.
  3. Repetition — რა იმეორება? რა ყოველთვის ახალი?
  4. Failure mode — სად ცდება? კონკრეტული example.
  5. Counterfactual — 1 საათი თუ დაგიზოგა, რას გააკეთებ? ("არაფერს" = save არ ღირს)
Test task default: Interview-ის ბოლოს 1-3 დღიანი test assignment ask. Dev decline → reject idea, არა push.
Domain-fit rule: კითხვები domain terms-ით, არა classic textbook terms-ით. "Schema / mapping" რომ ქართულ ERP-ის dev-ს ესმოდეს — ჯერ domain-ს გაეცანი.
Output: docs/JTBD_<domain>_<person>_<YYYY-MM-DD>.md

Slot 3 — Pre-mortem (Gary Klein)

Trigger: Build + Interview pass → სანამ ship/commit/recommend, 10-წთ pre-mortem.
Script: "წარმოვიდგინოთ: 3 თვე გავიდა. ეს აიდეა ჩავარდა. რა მოხდა?"
Categories: technical / adoption / capacity / economic / external.
Rule: 5+ failure cause. 2+ cause-ს mitigation თუ არ აქვს → don't ship, redesign.
Output: docs/PREMORTEM_<idea>_<YYYY-MM-DD>.md

Gate summary

SlotArtifactPass criterion
BMLWorking prototype / test resultBaseline vs target pre-defined, outcome measured
JTBDInterview file + test task outcome≥1 dev/user agreed test task OR idea rejected
Pre-mortem5+ failure modes, mitigatedNo orphan failure mode
All 3 pass → PRIORITY OK. Any fail → BLACKBOX / UNTESTED.
