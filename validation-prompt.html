Tier system NEW v2

TierროდისSlots საჭიროდრო

MINIMedium: single-channel test, internal tool, reversible hypothesis, low cost-of-failureBML + 1 complement (JTBD or Pre-mortem)4-8 სთ

Tier downgrade forbidden mid-validation (no convenience-driven scope reduction). Heavy → Mini გადასვლა მხოლოდ explicit owner approval-ით.
Tier escalation OK: Mini-ში material risk discovered → escalate Heavy.

"Priority" threshold NEW v2

  • Heavy: all 3 slots PASS + Viktor/peer ACK + owner sign-off (if owner-facing)
  • Mini: BML PASS + selected complement PASS + EP0 ≥7
  • Skip: OKR-KR trace explicit + EP0 ≥8 + owner go

Below threshold → label = hypothesis / blackbox / untested / idea. არასოდეს "priority".

Slot order — JTBD → BML → Pre-mortem REORDERED v2

v1 was BML → JTBD → Pre-mortem. v2 puts JTBD first — domain-ground before building. Build without domain understanding = textbook artifact, არა Bivision-fit (Apr 23 Gurafa lesson).

Slot 1 — Jobs-To-Be-Done (Christensen)

Pre-interview domain check: domain terminology verify (real terms, not textbook), existing artifacts read, peer-challenge interview script before send.
5-Q (15 წთ): Context · Pain · Repetition · Failure mode · Counterfactual
Test task default: 1-3 დღე ask. Decline → reject idea, არა push.
Output: docs/JTBD_<domain>_<person>_<date>.md

Slot 2 — Build-Measure-Learn (Ries)

Build: minimum working artifact 1-3 დღე. Code/measurement, არა slide.
Measure: ციფრი/binary outcome pre-defined. Numeric sanity check (95K vs 950K typo prevention).
Learn: pass/fail ორივე actionable.
Output: docs/BML_<idea>_<date>.md

Slot 3 — Pre-mortem cost-symmetric (Klein) UPDATED v2

Side A — cost of doing wrong: "3 თვე გავიდა, ჩავარდა — რა მოხდა?" Min 5 failure mode + mitigation.
Side B — cost of NOT doing NEW v2: "3 თვე გავიდა, არ გავაკეთეთ — რა დავკარგეთ?" Min 3 not-doing costs + magnitude.
Decision: A > B → don't ship; B > A → ship even with high A; A ≈ B → Mini scope.
Output: docs/PREMORTEM_<idea>_<date>.md

Expiration / refresh rule NEW v2

TierDefault expirationRefresh trigger (any)
Heavy30 დღეOwner directive change · market shift · team change · metric drift · competitor move
Mini14 დღეOutcome metric drift · scope expansion · repeated failure
SkipNoneOKR re-prioritization (re-check every session)

Refresh ≠ re-run all 3. Refresh = relevant slot only. Stamp every validation file: validated-at · expires · tier.

Anti-patterns (forbidden)

  1. "Priority" label below tier threshold
  2. Tier downgrade mid-validation (convenience reduction)
  3. JTBD skipped on user/dev claim
  4. Pre-mortem one-sided (only A, no B)
  5. Validation older than expiration without refresh
  6. "Test it" without baseline + target pre-defined
  7. Numeric claims (revenue, %, count) without order-of-magnitude sanity check
