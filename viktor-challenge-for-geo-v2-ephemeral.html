2. Technical calibrations (surgical)

2.1 TTFB diagnosis — mixed signal

Audit attributes TTFB 1.35s partly to "LiteSpeed Cache Mobile OFF". Calibration: Mobile Cache affects render/LCP, not TTFB (TTFB = server first-byte time, happens before cache layer return). Your P1 Redis/Memcached recommendation correctly targets TTFB. Suggest decoupling the two in the copy:
  • TTFB root cause → PHP opcache + object cache (Redis/Memcached) + DB (MariaDB upgrade)
  • LCP mobile gap → LS Cache Mobile ON + QUIC.cloud CDN
Cleaner diagnostic chain, same actions.

2.2 2FA threat framing

Line: "info@bivision.ge is publicly visible → brute-force risk high". Technical nuance: info@bivision.ge is a contact address — WordPress admin login takes WP username, not email. Probe: curl -s https://bivision.ge/?author=1 exposes real username via redirect. That's the actual enumerable surface, and the real case for 2FA. Suggest rephrasing to the author-enumeration vector — same recommendation, sharper logic.

2.3 Baseline-math gap (biggest structural note)

Projections like "+5-15% mobile conv", "+10-30% organic ranking lift", "+5-15% AI-citation referrals" cite generic benchmarks (Unbounce, Portent) but are not anchored to current Bivision numbers. Without current-state conversion %, organic sessions, and MQL count, the deltas can't be sized in absolute impact. Suggestion:
  • Pull GA4 (last 30 day): sessions, mobile share, conversion rate
  • Pull GSC (last 28 day): impressions, clicks, avg position, top 20 queries
  • Rank Math Analytics: backlink count, referring domains
Then restate projection as current_metric × range, not bare %. Changes nothing about the recommendations — only how the impact is sized. Rank Math Analytics is already connected per your stack inventory, so this is a 30-minute pull, not a new dependency.

2.4 "llms.txt +5-15% AI-citation referrals"

The channel is real and emerging, but the number range is speculative for the Georgian B2B BI market (no measured Perplexity/ChatGPT referral baseline exists publicly). Suggest flagging the range as [HYPOTHESIS] rather than projecting it alongside benchmarked items. Ship llms.txt anyway — the cost is trivial and the positioning is correct.
